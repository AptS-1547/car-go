# Code of Conduct

car-go is a technical open-source project. We want issues, pull requests, reviews, and discussions to remain respectful, focused, and useful.

## Expected Behavior

- Be respectful when disagreeing.
- Give specific, actionable feedback tied to the code or observed behavior.
- Assume good intent while asking for evidence and clarification.
- Keep reports reproducible and discussions on topic.
- Respect privacy, security disclosures, and other contributors' boundaries.

## Unacceptable Behavior

The following is not acceptable in project spaces:

- harassment, discrimination, hate speech, threats, or personal attacks
- sexualized or unwanted attention, doxxing, or sharing private information
- impersonation, fraud, spam, or deliberately misleading reports
- trolling, baiting, or repeatedly derailing technical discussions
- publishing private vulnerability details before maintainers can respond

## Scope and Reporting

This policy applies to GitHub issues, pull requests, discussions, review comments, documentation, and other spaces used to represent car-go.

Report conduct concerns privately to `report@esaps.net`. Include links, screenshots, timestamps, and a concise description when available. Do not use a public issue for harassment reports or private information.

Maintainers may edit or remove content, lock discussions, reject contributions, or restrict participation when necessary to keep project spaces usable. Reports will be reviewed in good faith and personal information will be handled as confidentially as reasonably possible.
